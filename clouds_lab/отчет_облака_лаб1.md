Алгоритм работы: Сопоставить входящие данные от провайдера с его же документацией. Написать в соответствие колонкам справа значения 5 колонок слева, которые бы однозначно классифицировали тип сервиса. Для столбцов IT Tower и Service Family значения можно выбрать из образца.

Ну начнем с того, что нужно в целом узнать что такое AWS - это облачная платформа, предоставляющая более 200 
полностью управляемых сервисов для вычислений, хранения, баз данных, сетей, аналитики, машинного обучения 
и уведомлений. AWS использует модель pay-as-you-go (плати за использование) с различными типами потребления:
по часам (compute), по объёму (storage), по запросам (API), по байтам (data transfer). Сервисы классифицируются 
по уровням абстракции — от IaaS (инфраструктура) до SaaS (приложения)

Методология классификации:
- IT-Tower: Высокий уровень (Storage/Network/Database/Application)
- Service Family: Категория сервиса (Cloud Storage, Data Warehouse, Messaging)
- Service Type: Конкретный сервис (Object Storage, Virtual Private Cloud)
- Service Sub Type: Тип потребления (Data Transfer, Compute Nodes, API Requests)
- Service Usage Type: Точный billing-метрик (DataTransfer-Out-Bytes, Node Hours)

Источники: AWS Pricing docs, CUR Usage Type guides
